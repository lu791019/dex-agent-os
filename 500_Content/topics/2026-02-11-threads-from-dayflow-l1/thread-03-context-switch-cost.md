我今天開了 24 個 Claude Code session

在 9 個以上的工作串之間跳來跳去
最長一段專注時間只有 25 分鐘

看起來超忙超高產對吧

但說實話
我不確定今天真正「完成」了幾件事

這就是 context switch 最可怕的地方
不是你做得少
是你做了很多 卻沒有一件做到位

每次切換任務 大腦需要 10-15 分鐘重新載入上下文
一天切 20 次
光是「重新想起我剛剛在幹嘛」就燒掉好幾個小時

更慘的是 LINE

商務跟社交混在同一個介面
999+ 未讀在那邊跳
手指不自覺就滑過去了

然後你就從寫程式的心流
掉進「欸這個群組在聊什麼」的黑洞

高產出 ≠ 高效率
忙碌感 ≠ 進展感

我現在逼自己一件事：
每天至少要有一段 90 分鐘不被打斷的時間
關通知 關 LINE 什麼都不切

比起同時做十件事
不如把一件事做完再說

你今天最長的一段專注時間是多久？
