AI 寫程式最危險的不是寫出 bug，是寫出你看不懂的正確答案。

這就是「認知債務」——你「有了」程式碼，但你沒有「懂了」。

跟技術債不一樣，技術債你看得到，程式碼亂、架構醜，你知道要還。

認知債你看不到。系統跑得好好的，直到某天壞了，你才發現團隊裡沒有人能解釋當初為什麼這樣設計。

我自己用 AI 開發 side project，回頭翻 code 的時候就有這種感覺——這段邏輯是對的，但我說不出為什麼選這個方案而不是另一個。

人類理解事物需要一個迴圈：碰壁 → 猜測 → 驗證 → 修正。AI 直接幫你跳到終點，跳過的那段路，就是你欠下的債。

像五條悟的無量空處，一口氣把無限資訊灌進你腦中，你的思考根本跟不上。

差別只在於——五條悟是攻擊，AI 是在「幫」你。

留言處繼續聊：你有沒有遇過「AI 寫的 code 跑得動但你其實不太確定為什麼」的時刻？
