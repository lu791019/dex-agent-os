# 每日素材捕捉規則

當 Dex 輸入的訊息以下列標籤開頭時，自動追加到當日素材池檔案。

## 辨識標籤

| 標籤 | 歸入 section | 說明 |
|------|-------------|------|
| `會議：` | `## 會議` | 會議重點、決策、待辦 |
| `經驗：` | `## 經驗` | 工作中的實戰經驗 |
| `心得：` | `## 心得` | 閱讀、對話、思考後的感想 |
| `靈感：` | `## 靈感` | 內容主題、產品點子、寫作方向 |
| `筆記：` | `## 筆記` | 其他值得記下的事 |

## 行為

1. 偵測到標籤 → 取得今日日期（`YYYY-MM-DD`）
2. 素材池路徑：`000_Inbox/daily/YYYY-MM-DD.md`
3. 檔案不存在 → 自動建立：
   ```markdown
   ---
   date: YYYY-MM-DD
   ---
   # 每日素材 — YYYY-MM-DD
   ```
4. 將內容追加到對應的 `## section`（section 不存在則建立）
5. 每條前綴 `- `，保留原文
6. 回覆一行確認：「已記錄到 [section]」

## 觸發詞捷徑

當 Dex 輸入以下單一詞彙時，視為觸發對應 skill：

| 觸發詞 | 對應 skill | 說明 |
|--------|-----------|------|
| `轉譯` | `/轉譯` | 素材池 → Threads 草稿 |
| `回顧` | `/回顧` | 快速回顧今天 |
| `記住` | `/記住` | 萃取學習到 memory |
| `模板` | `/模板` | 儲存寫文框架 |

判斷條件：訊息僅包含觸發詞本身（可有前後空白），或觸發詞後接具體內容（如「轉譯 用這個角度...」）。

## 素材池與 readings 的關係

- 素材池（`000_Inbox/daily/DATE.md`）= 手動輸入的想法和經驗
- Readings（`000_Inbox/readings/DATE-*.md`）= 自動匯入的閱讀素材
- `/轉譯` 和 `/daily-all` 都會同時讀取兩者，不需複製資料
