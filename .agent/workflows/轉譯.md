# 轉譯 — 素材即時轉 Threads 草稿

從今日素材池 + readings + 近期 insights 中挑選角度，產出 Threads 草稿，直接印在對話中。

## 觸發方式

- IDE 內：`/轉譯`
- 觸發詞：`轉譯`（見 `30-daily-capture.md`）
- 可附帶內容：`/轉譯 我覺得 AI 工具最重要的不是工具本身...`

## Step 0：判斷模式

- **有種子模式**：Dex 在 `/轉譯` 之前（同一則訊息或前幾則）寫了一段自由文字 → 以這段文字為主要種子，跳到 Step 2 直接寫稿
- **瀏覽模式**：Dex 只打 `/轉譯`，沒附帶內容 → 走完整的素材挑選流程

## 共通步驟

### Step 1：讀素材

依序讀取以下來源（有什麼讀什麼，缺的跳過）：

1. `000_Inbox/daily/DATE.md`（今日素材池）
2. `000_Inbox/readings/DATE-*.md`（今日自動匯入的閱讀）
3. `510_Insights/`（近 3 天的 insight 檔案）

### Step 2：讀風格

1. `800_System/references/style-dna/threads-dna.md`
2. `canonical/rules/10-writing-style.md`（「改寫與創作流程」section）

### Step 3：讀框架（備用）

1. `800_System/references/writing-frameworks.md`（不主動列出，Dex 指定時才用）

---

## 有種子模式（Dex 附帶了文字）

### Step 4s：找相關素材

用 Dex 的文字作為核心角度，從素材池 + readings 找到相關素材來豐富內容。

### Step 5s：簡要確認方向

> 我打算用 [角度] 結合 [相關素材]，這樣寫？

Dex 確認 → 直接進 Step 7 寫稿。

---

## 瀏覽模式（Dex 沒附帶文字）

### Step 4b：提出角度

從素材中提出 3-5 個角度，優先「個人經驗 x 文章」的交叉點。

格式：
```
1. [角度名] — [一句話說明切入點]
2. [角度名] — [一句話說明切入點]
3. ...
```

### Step 5b：Dex 選擇

Dex 選 1-2 個角度。可選附加：
- 指定框架（如「用框架 1」）
- 給模板（直接貼結構）
- 給參考 URL（用 WebFetch 抓取分析）

### Step 6b：選框架

- Dex 指定了框架 → 用指定的
- Dex 沒指定 → 根據角度自動選最適合的 DNA 模式或已有框架

---

## Step 7：產出草稿

產出三種格式：

### 7a. Threads 草稿（主要輸出 — 直接印在對話中）

遵循 threads-dna.md 的風格，結構：主文 + 串文（內容密度不夠就一條打完）。

**印出格式**：
```
---
**主文：**

（主文內容）

---

**串文：**

（串文內容，如果有的話）
```

### 7b. Facebook 貼文草稿（存檔，不印）

同一主題的 Facebook 版本，語氣可以更完整、更長。

### 7c. 短影音腳本（存檔，不印）

同一主題的短影音腳本版本（Hook → 正文 → CTA，60-90 秒）。

## Step 8：Dex 確認或修改

針對 Threads 草稿進行互動修改，直到 Dex 滿意。

## Step 9：存檔

- `530_Channels/threads/DATE/<slug>.md`
- `530_Channels/facebook/DATE/<slug>.md`
- `530_Channels/short-video/DATE/<slug>.md`

使用既有 frontmatter 格式：
```yaml
---
topic: <slug>
channel: threads  # 或 facebook / short-video
status: draft
created: DATE
---
```

## Step 10：框架存檔（可選）

寫完後問 Dex：「這個寫法要存到 writing-frameworks.md 嗎？」

確認 → 追加到 `800_System/references/writing-frameworks.md`，格式：結構 + 邏輯 + 適用場景 + 範例。

---

## 框架與範例輸入方式

1. **直接給框架**：`/轉譯 用這個框架：呼喚你的TA，給實用建議...` → 解析結構套用
2. **給參考 URL**：`/轉譯 參考 https://www.threads.com/@xxx/post/xxx` → WebFetch 抓取分析結構
3. **引用既有框架**：Dex 說「用框架 1」→ 從 writing-frameworks.md 取用

## 寫稿原則（速查）

1. 重新詮釋，不是翻譯 — 讀完素材後問「跟朋友講第一句會說什麼？」
2. 開場先共感，再翻轉
3. 引導式 > 判定式
4. 條列 = 粗體主張 + 口語展開
5. 收尾只留一句能截圖的話
6. 主文 + 串文結構（密度不夠就一條打完）
7. 不用 emoji，靠換行和節奏製造可讀性
8. 抽象描述要錨定一個具體畫面
