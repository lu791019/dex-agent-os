# 模板 — 儲存寫文框架

將 Dex 提供的框架、範例、結構存到 writing-frameworks.md，供 `/轉譯` 時引用。

## 觸發方式

- IDE 內：`/模板`
- 觸發詞：`模板`（見 `30-daily-capture.md`）
- 可附帶內容：`/模板 呼喚你的TA → 共感痛點 → 3 條實用建議 → 金句收尾`

## 執行步驟

### Step 1：接收框架內容

Dex 可能提供：
- 純文字描述框架結構
- 參考 URL（Threads 貼文、文章等）
- 兩者皆有

### Step 2：分析結構

- 如果有參考 URL → WebFetch 抓取貼文內容，分析其結構和風格
- 從 Dex 的描述 + 參考貼文中提煉出：
  - **結構**：段落 / 節奏安排
  - **邏輯**：論述推進方式
  - **適用場景**：什麼主題適合用這個框架
  - **範例**：從參考貼文擷取或 Dex 提供

### Step 3：格式化

讀取 `800_System/references/writing-frameworks.md` 確認現有格式，產出標準化框架：

```markdown
## N. [框架名稱]

**結構**

（結構描述或模板）

**邏輯**：（一段話說明論述推進方式）

**適用場景**
- ...

**範例**

> （範例內容）

**變體**
- ...（如果有的話）
```

### Step 4：確認

印出格式化結果讓 Dex 確認或修改。

### Step 5：存檔

追加到 `800_System/references/writing-frameworks.md` 的 `<!-- 新框架請加在這裡 -->` 標記之前。

### Step 6：回報

回覆框架編號，供 `/轉譯` 時引用：「已存為框架 N，轉譯時可說『用框架 N』。」

## 原則

- 框架要夠抽象可重用，不是只記錄一篇貼文
- 一個框架對應一種論述推進模式
- 範例盡量用 Dex 自己的貼文或改寫
