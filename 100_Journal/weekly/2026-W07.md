## 學到什麼
- **Meta 個人開發者用「用戶權杖產生器」遠比 OAuth 簡單**：localhost redirect URI 被 Meta 全面封殺，先查平台限制文件再動手，可省去 3 輪無效嘗試
- **Style DNA 萃取效果超出預期**：78 篇真實範例的 7 維度分析精準捕捉寫作風格，比手動整理品質高很多——LLM 在風格辨識上已經可用
- **AgentSkill 設計哲學核心**：協議與人格分離（Secretary Core）、原則導向優於功能導向、測試三問「AI 能找到/理解/解決嗎？」、Level 2/3 用 Skill 觸發 Skill 的架構——Skill 不是工具，是知識管理的結構化
- **Patchright 不能與已開啟的 Chrome 共存**，NotebookLM 認證有效期僅 7 天，自動化腳本對瀏覽器狀態的依賴是整合的隱形成本
- **「先規劃再動手」有量化證據**：Phase 3 從 plan mode 確認到端到端測試通過只花一個下午，對比邊做邊改的經驗差異巨大

## 模式與趨勢

### 🔀 高頻上下文切換（連續兩天出現）
兩天都出現每 10-15 分鐘就在不同任務間跳轉的行為。02/09 是「HR 表單 ↔ 面試 ↔ Phase 3 ↔ 個人瀏覽」，02/10 是「影片 ↔ 安裝工具 ↔ 社群 ↔ 筆記 ↔ 訂房」。專注品質分別只有 3/5 和 2.5/5，與 CLAUDE.md 中「單一時段專注一個 session」的原則直接矛盾。

### 🧲 工具收集傾向 > 工具深度使用
02/10 一個下午安裝/研究了 6+ 個工具（Supabase、NotebookLM、OpenSpec、Context7、Spectra、WailBrew），但實際深度使用的只有 NotebookLM。這不是第一次出現——「磨刀」和「砍柴」的比例需要有意識地控制。

### 📉 產出集中在週初，後段斷裂
02/09 是高產日（Phase 3 核心實作一氣呵成），02/10 轉入研究模式後零程式碼產出，02/11-15 完全沒有日記記錄。兩天的資料顯示一個清楚的模式：**開發日和研究日交替出現，但研究日缺乏收斂機制**。

### 📝 筆記系統分散
同時使用 Google Doc、NotebookLM、ChatGPT、Notion 四處做筆記，但這些內容沒有回流到 dex-agent-os 的知識體系（`300_Learning/` 或 `800_System/knowledge/`）。知識萃取系統是自己建的，卻沒有在自己的工作流中閉環。

### 🕐 有效工作時間偏短
兩天都只有約 3.5 小時有效活動，且被個人事務（HR 表單、訂房、面試溝通）切走 20-30%。午間都有大段空白（02/09 五小時、02/10 上午無記錄）。

## 卡住的地方
- **OAuth redirect URI 連環失敗**：Meta 封殺 localhost，花 20 分鐘撞牆後才轉向用戶權杖產生器——根因是沒先讀平台限制文件
- **Rate limit 反覆觸發**：多 session 切換 + 高 token 消耗，每日內容管線首次觸發未能完成
- **技術債堆積**：Threads API token 60 天過期需手動更新、collector 無 rate limiting 保護、NotebookLM OAuth 7 天過期需定期重新認證
- **02/11-15 完全沒有日記**：無法評估這五天的狀態、產出和問題。這本身就是最大的卡點——系統建了但沒有持續使用
- **Phase 3 feature branch 合併時機未定**：文件更新和 commit 停留在 working tree，有遺失風險

## 下週重點
1. **恢復每日日記習慣**：本週後 5 天的空白讓回顧幾乎無法進行，下週至少記錄 5/7 天，哪怕只寫「今日一句話 + 做了什麼 + 能量」三個欄位
2. **Phase 3 收尾閉環**：合併 feature branch、設定 Threads token 過期提醒、補上 rate limiting——收尾優先於開新坑
3. **AgentSkill 學習筆記歸檔**：02/10 的研究成果散落在 Google Doc / NotebookLM，花 30 分鐘整理進 `300_Learning/`，讓知識回流到自己的系統
4. **研究日加收斂機制**：每次研究結束前強制寫「3 個關鍵收穫 + 1 個下一步行動」，避免螺旋式學習但不收斂
