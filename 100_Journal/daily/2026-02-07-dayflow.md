## 行為模式 & 洞察

### 🔀 高頻上下文切換
單一活動平均 15–20 分鐘，且多數時段同時開啟 3–5 個應用視窗。典型模式：Terminal 跑 agent → 切 Chrome 查資料 → 切 LINE 回訊息 → 切回 Terminal。這種碎片化切換降低了深度工作的品質。

### 🌙 深夜效率遞減
21:00 後的活動明顯碎片化：21:03–21:33 在社群/問卷/Facebook 間跳轉，23:25–00:12 花 47 分鐘瀏覽 Discord 但無實質產出（僅閱讀 onboarding 文字）。深夜時段更傾向「消費型瀏覽」而非「生產型工作」。

### 📦 「規劃完美主義」傾向
Job Hunter → Agent OS → flighttracker → consultation template，一個晚上觸及 4 個專案的規劃，但沒有任何一個推進到可運作的產出。目錄結構反覆調整（canonical → sync → triage），暗示對架構的完美追求可能拖慢實際進度。

### ⏳ 被動等待消耗時間
Rate limit 等待 + Antigravity 閒置 + token 清單靜觀 ≈ 30–40 分鐘的隱性浪費。這些空窗期沒有被有效利用（例如切換到離線任務）。
