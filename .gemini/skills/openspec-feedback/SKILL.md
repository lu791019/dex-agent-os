---

Help the user submit feedback about OpenSpec.

**Process**

1. **Gather context** from the conversation
2. **Draft enriched feedback** with clear title and body
3. **Anonymize sensitive information** (paths, keys, names)
4. **Present draft for approval**
5. **Submit on confirmation** using `openspec feedback "title" --body "body"`

**Guardrails**
- MUST show complete draft before submitting
- MUST ask for explicit approval
- MUST anonymize sensitive information
