
# 記住 — 萃取學習到 Memory

從今日素材池和日記中萃取值得長期記住的學習和反思，寫入 memory 檔案。

## 觸發方式

- IDE 內：`/記住`
- 觸發詞：`記住`（見 `30-daily-capture.md`）
- 可附帶內容：`/記住 今天學到 cursor agent 規則比寫 code 重要`

## 執行步驟

### Step 1：收集資料

讀取以下來源：

1. `000_Inbox/daily/DATE.md`（今日素材池）
2. `100_Journal/daily/DATE.md`（L2 精煉日記）
3. `~/.claude/projects/-Users-dex-dex-agent-os/memory/learnings.md`（現有 learnings）
4. `~/.claude/projects/-Users-dex-dex-agent-os/memory/reflections.md`（現有 reflections）

### Step 2：列出候選學習

**如果 Dex 附帶了具體內容** → 直接分類該內容，跳到 Step 3 確認。

**如果沒附帶** → 從素材池和日記中挑出候選，分類呈現：

```
### Learnings（工具/技術/方法）
1. [學習內容] — 來源：[素材池/日記]
2. ...

### Reflections（反思/原則/心態）
1. [反思內容] — 來源：[素材池/日記]
2. ...
```

### Step 3：Dex 選擇

讓 Dex 選要記住哪些（可全選、可挑選、可修改措辭）。

### Step 4：寫入 Memory

- Learnings → 追加到 `memory/learnings.md`
- Reflections → 追加到 `memory/reflections.md`

### Step 5：重複檢查

寫入前比對現有內容，如果已有語意相近的條目：
- 告知 Dex「這條跟現有的 [X] 很像，要合併還是都保留？」
- 合併 → 更新現有條目
- 保留 → 追加新條目

## 原則

- 只寫入被 Dex 明確確認的內容
- 保持 learnings.md ≤ 120 行、reflections.md ≤ 80 行
- 超出行數時，建議 Dex 歸檔舊條目到 `800_System/knowledge/`
- 語意重複的不重複寫入
