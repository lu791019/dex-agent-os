# 寫作框架收集

> 跨頻道的論述公式與文案結構。寫文前可翻閱，挑一個適合當前主題的框架套用。
> 持續累積中，每個框架附上：結構、適用場景、範例。

---

## 1. 不是錯覺框架

**結構**

```
如果你發現 [問題]，不是錯覺、也不是 [迷思]，是 [真相]：

- 建議 1
- 建議 2
- 建議 3
```

**邏輯**：先驗證讀者的感受（「你沒有想錯」），再破除常見誤解，最後給出真正的原因和行動建議。三段式：共感 → 破迷思 → 給方向。

**適用場景**
- 職涯焦慮類（升遷卡住、技能過時、薪資天花板）
- 技術認知類（工具選錯、架構誤區、效能迷思）
- 任何「讀者隱約感覺到問題但說不清楚」的主題

**範例**

> 如果你升到 Senior 之後開始覺得卡住，不是錯覺，也不是技術退步，是你還沒意識到一件事：你的角色已經變了。
>
> - 不要只看技術，要開始看「成本」
> - 不要只記住系統，要理解「設計原則」
> - 開始成為成本的守門人

**變體**
- 可以把「建議」換成「觀察」或「真相的不同面向」
- 收尾可加一句金句或開放提問
- 條列數量 3-5 條皆可，不要超過 5 條

## 2. 里程碑轉推廣框架

**結構**

```
[里程碑文] — 用一行一階段的時間軸呈現個人歷程，有高有低

[推廣文] — 從里程碑的某個轉折點，自然帶出課程/產品：
1. 回扣里程碑中的具體經歷（「我當初...」）
2. 用「如果重來 / 回頭看」翻轉成推薦理由
3. 課程細節融入敘事，不用條列規格
4. 「我自己就是講師」用「我把經歷整理成一門課」帶出
5. 收尾：溫暖但不替讀者決定（「你不用一個人摸」）
6. 折扣碼 + CTA 壓在最後一行，一句帶過
```

**邏輯**：先用里程碑建立「過來人」的信任感，再用自身經歷的痛點（走冤枉路、方向不明）作為橋樑，推薦就變成「幫你少走我走過的路」而不是「快來買」。

**適用場景**
- 課程推廣搭配個人故事
- 產品推薦需要實證基礎時
- 任何「我用過 / 我做的 / 我教的」類推廣

**關鍵原則**
- 條列課程規格 = 廣告感，融入敘事 = 朋友推薦
- 「我是講師」直說太硬，「我整理成一門課」自然帶出
- 數據（市場成長 40%）可省略，個人經驗的說服力更強
- 折扣碼不要獨立段落強調，跟連結 CTA 合併一行
- 收尾語氣：溫暖陪伴 > 硬推行動，不替讀者做決定

**範例收尾**

> 我沒辦法替你決定要不要轉，應該說，沒有人可以。但如果你現在跟當年的我一樣，在現有的工作裡覺得悶、趁空檔偷學、不確定方向對不對——至少這條路，你不用一個人摸。
