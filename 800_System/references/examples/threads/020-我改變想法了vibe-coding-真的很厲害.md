不過別急著嗆我，我說的可不是那種靠「氛圍」來寫程式的方法

而是靠「規格」下指令，讓 AI 照流程自動幫你完工
這才是真正的 AI Coding

或者說：AI 時代下軟體工程師的該有的「AI Coding」方法論

不是被 AI 取代，而是學會讓 AI 為你工作，軟體思維則會讓你無可取代

很多人用 Vibe coding 的方式，其實都不對：
下 prompt， 然後AI 寫爛， 複製程式，再 下Prompt

結果你就只是在當 AI 的保母

但如果你換個方法，讓 AI 配合獨有的技術思維- BDD

你只要定義需求，AI 就能跑完整個開發，並完成測試和修正

這就是水球軟體學院(WSA)要開設的工作坊

你會學到：

- 把需求拆解成 Feature File
- 讓 AI 自動生成執行規格
- 自動實作、自動測試、自動修正
- 將軟體技術思維真正導入開發流程

你寫的不再是 prompt，而是工程師級別的規格書
