「先規劃再動手」這句話聽起來像廢話對吧。

但我昨天有了量化證據。

Phase 3 的內容生產管線——從 API 串接、風格萃取、到主題生成、草稿輸出，四個模組端到端。

從 plan mode 確認架構到全部測試通過，一個下午。

同樣複雜度的東西，之前邊寫邊改，搞了快兩天還在 debug。

差別不是我變強了。
是我花了 30 分鐘先把 implementation_plan.md 和 task.md 寫好。

想清楚每個模組的輸入輸出、資料流向、依賴關係。

動手的時候腦子不用同時處理「要做什麼」跟「怎麼做」。

少了那層認知負荷，寫碼速度直接翻倍。

你下次開工前，願意多花 30 分鐘寫計畫嗎？
