用戶已經在 prompt 中提供了完整的主題內容、風格 DNA 和寫作規則，我不需要再讀取檔案。直接根據提供的素材產出 Threads 貼文。

你以為寫設計文件是在浪費時間，其實它才是你開發速度的加速器。

我今天 21 分鐘跑完 5 個 commit，從頭到尾零卡關。不是因為我打字特別快，是因為寫 code 之前，所有架構決策都已經在設計文件裡定案了。

寫 code 的時候同時做架構決策，就像一邊開車一邊看地圖——你以為自己在前進，其實一直在繞路。

不寫設計文件的隱性成本：

- 邊寫邊改架構，一個 PR 拖三天還沒 merge
- code review 來回五六輪，其實在討論的是「一開始就該想清楚的事」
- 上線後才發現設計有洞，hotfix 比原本的開發還花時間

設計文件不是官僚流程，是把「猶豫」從實作階段提前抽掉。

當你寫 code 的時候腦袋只剩下執行，那個速度感，試過一次就回不去了。
