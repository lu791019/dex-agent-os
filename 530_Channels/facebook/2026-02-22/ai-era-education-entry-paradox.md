AI 讓寫程式的門檻低到不行，但找到工程師工作的門檻反而變高了。

這個矛盾聽起來反直覺，但你想一下就會發現它很合理。當 ChatGPT 能幫任何人產出能跑的程式碼，「會寫程式」就不再是一種稀缺能力。企業不缺會下 prompt 的人，他們缺的是能 debug、能設計架構、能在凌晨三點扛住生產環境出包的人。

問題出在哪？出在入門跟就業之間的距離被拉開了。

以前初階開發者有自己的生存空間——簡單的 CRUD、前端切版、基本的 API 串接。現在這些工作正在被「AI 輔助的資深開發者」一個人吞掉。不是職缺消失了，是同一份工作要求的能力密度變高了。Junior 的位置被壓縮，不是因為你不夠好，是因為遊戲規則改了。

這讓我覺得，教人寫程式的人現在背負著一個比以前更重的責任。

當你告訴一個人「學完 Python 就能轉職」，你賣的不是課程，是一個期望。如果這個期望跟市場現實嚴重脫節，受傷的是那個信任你、花了半年時間全力投入的學生。畫大餅短期確實賣得動，但長期來看，信譽會崩。反而是願意說實話的人——告訴你這條路現在長什麼樣、哪些地方變窄了、怎麼走比較穩——建立了更深的信任。

那學 coding 到底還有沒有用？我覺得有，但目的要調整。

學程式是為了理解技術怎麼運作、提升你拆解問題的能力，不是為了保證拿到一份 offer。把它當擴展能力的工具，別當進入某個行業的船票。這個心態轉換，可能比學會任何一個框架都重要。

你身邊有朋友正在考慮轉職寫程式嗎？如果他來問你的意見，你會怎麼跟他說？
