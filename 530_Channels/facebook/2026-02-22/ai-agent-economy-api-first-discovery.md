你的下一個客戶，可能不是人類。

前陣子看到一個案例讓我停下來想了很久：WorkOS 跟 Claude 的整合，讓 AI Agent 可以自動完成 auth 串接——從發現服務、評估 API 規格、到實際整合，整個流程裡面沒有任何一個人類 sales 介入。

這不是概念驗證，是已經在跑的東西。

仔細想想，這件事的意義比表面上大很多。傳統 B2B 的獲客漏斗——行銷頁面、cold email、webinar、demo call——每一層都是設計給人類看的。但 AI Agent 不看你的 landing page 寫得多漂亮，不會回你的開發者關係信，更不會花 30 分鐘聽你的產品 demo。

它需要的是：結構化的 API spec、機器可讀的定價、可程式化的試用流程。

你精心設計的 sales funnel，對 Agent 來說每一層都是摩擦力。

這件事正在催生一整套新的基礎建設。OpenAPI spec 讓你的服務能被機器理解，Schema.org 標記讓你在語意網路上被發現，MCP（Model Context Protocol）則是讓 AI Agent 直接跟你的服務對話的協定。這不是三五年後的事——這些標準現在就在被採用。

對技術人來說，這裡有一個結構性的機會：資料工程師和後端開發者天生就離這個趨勢最近。你每天在做的事——API 設計、資料結構化、服務治理——就是 AI Agent 經濟裡最核心的能力。差別只在於，你有沒有意識到你正在為誰建造這些東西。

Anthropic CEO 預測 AGI 一到三年內到來。不管你信不信這個時間表，有一件事很確定：先把 API-first 基礎建設搞好的公司，會拿到結構性優勢。而這個窗口期，比多數人想像的短。

你現在負責的服務或產品，如果明天要「賣」給 AI Agent，最大的障礙會是什麼？
