# 先賣再做——自由工作者該不該把未完成的產品寫進報價？

## 核心論點
把還沒做完的產品寫進提案不是詐欺，而是一種「用市場驗證倒逼交付」的策略——但前提是你有能力在承諾期限內兌現，否則省下的開發時間會全部變成信譽負債。

## 關鍵素材
1. **真實案例**：HireReady/JobFit 還在開發階段就包進客戶報價，提升了提案的感知價值與專業度，但同時背負了必須如期交付的壓力
2. **先賣再做的好處**：提前驗證市場需求（避免做完沒人要）、增加報價的產品化感（不只是工時買賣）、用外部 deadline 推動自己完成 side project
3. **先賣再做的風險**：交付延遲損害信譽、品質妥協（趕工出貨）、客戶期待管理失控（以為是成熟產品）
4. **MVP 思維的連結**：這本質上就是 Lean Startup 的「先賣再造」——用最小可行提案測試市場反應，差別在自由工作者賭的是個人品牌
5. **產品化策略的邊界**：從接案走向產品化的過渡期，「把服務包裝成產品」和「把半成品當成品賣」之間的界線在哪裡？
