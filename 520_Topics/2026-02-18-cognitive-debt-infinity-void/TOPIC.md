# 認知債務是 AI 時代的無量空處

## 核心論點
AI 輔助開發像五條悟的無量空處——一口氣把知識灌進腦中，你「有了」答案但沒有「懂了」，因為人類理解需要「碰壁 → 假設 → 驗證 → 修正」的漸進迴圈，而 AI 直接跳過了這個迴圈。

## 關鍵素材

1. **Simon Willison 的認知債務概念**：AI 加速開發讓開發者失去對系統的心智模型，團隊無法解釋設計決策時比 code 混亂更致命
2. **Steve Yegge 的觀察**：AI 把簡單工作自動化後，剩下的全是高認知負荷決策，每天 4 小時是實際可持續節奏
3. **技術債 vs 認知債的區分**：技術債看得到（程式碼亂），認知債看不到（說不清設計決策），後者更危險
4. **無量空處比喻**：認知債務 ≠ 資訊過載。資訊過載是量太大，認知債務是「灌入方式」跟人類理解方式根本不匹配
5. **有意識 vs 無意識借貸**：「知道自己哪裡沒吃透」是可控風險，「不知道自己不懂」才是定時炸彈。「先跑通再調整」是用實踐補回被跳過的理解迴圈

## 頻道適合度

- **Threads**：非常適合。用無量空處的比喻開頭自帶畫面感，動漫梗 + 技術洞察的混搭在 Threads 上很容易引發互動。一則講比喻、一則講自身經驗、一則講區分，3 則串就夠
- **Blog**：非常適合。可以展開成深度文——「技術債 vs 認知債」的對比框架、引用 Simon Willison 和 Steve Yegge 的原文、加入自己開發 Agent OS 的實際案例。字數空間足夠把論證講清楚
- **Newsletter**：適合。可以結合實踐建議（Phase 交界設檢查點、有意識借貸的心法），給讀者可帶走的框架
- **Facebook**：普通。概念偏技術社群，FB 受眾較泛，除非搭配更通俗的切入角度
- **ShortVideo**：有潛力但執行難度高。無量空處的動畫片段 + 旁白解說認知債務，視覺衝擊力強但需要素材剪輯
- **Podcast**：適合作為某集的段落素材，但單獨撐一集偏薄
